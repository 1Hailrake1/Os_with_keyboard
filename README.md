# Os_with_keyboard

**Os_with_keyboard** — это экспериментальная самописная операционная система, разработанная для изучения низкоуровневого программирования, механизмов загрузки, базового ввода с клавиатуры и взаимодействия с аппаратурой на уровне BIOS/CPU.  
Проект написан на **Assembly (NASM)** и **C**, содержит минимальное ядро и обработчики ввода, собирается через `Makefile` и запускается в QEMU.

---

## Цели проекта

- Изучение основ работы операционных систем.
- Практика разработки собственного загрузочного сектора.
- Получение навыков работы с BIOS-прерываниями, обработчиками клавиатуры и низкоуровневым вводом/выводом.
- Сборка и запуск bare-metal среды без зависимости от хостовой ОС.
- Выстраивание минимальной архитектуры ядра с поддержкой пользовательского ввода.

---

## Возможности

- Загрузка собственного ядра с нуля.
- Обработка нажатий клавиш через аппаратные прерывания.
- Вывод символов на экран (текстовый режим).
- Базовая работа с памятью на низком уровне.
- Простая циклическая система событий (kernel loop).
- Чистая структура проекта с разделением boot → kernel → drivers.

---

## Требования

Для сборки и запуска проекта нужны:

- **NASM** — ассемблер
- **GCC / Clang** (кросс-компилятор не обязателен, если код компилируется под i386)
- **Make**
- **QEMU** (рекомендуемый эмулятор)

Установка (Ubuntu/Debian):

```bash
sudo apt update
sudo apt install nasm gcc make qemu-system-x86
Сборка проекта
В корне репозитория выполнить:

make
После сборки в папке build/ появится файл:


os.bin
Это итоговый бинарник ОС, готовый для запуска в эмуляторе.

Запуск ОС
Запуск в QEMU:

bash
Копировать код
qemu-system-x86_64 -drive format=raw,file=build/os.bin
Либо, для более точной эмуляции real-mode:

bash
qemu-system-i386 -fda build/os.bin
